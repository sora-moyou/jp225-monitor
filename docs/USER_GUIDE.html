● Windows 10 / 11 · 無料 · v0.4.22

急変を 30 秒で
AI が日本語で説明する。

日経 225 先物 (OSE mini) の価格変動をリアルタイムで監視。直近のボラティリティを基準に「いまの相場として異常か」を判定。 その後 AI が、急変の原因をニュースや他銘柄から考えて、説明します。価格は 24/7 デーモンで蓄積し、起動時に即ウォームアップ。

3 ステップ

3 分でセットアップ

ダウンロードから初回起動まで。API キーは無料で取得できます。

1

ダウンロード

GitHub Releases ページ を開きます。

📥 Releases ページの見方

ページを開いただけでは .exe は見えません。下にスクロール して Assets セクションを探してください。

  1. リリース説明の下、▶ Assets という折り畳みを クリックして展開
  2. 4 つのファイルが表示されるので、その中の 📦 JP225.Monitor_X.X.X_x64-setup.exe をクリック

※ .sig や latest.json はアプリ側が自動更新で使うもので、利用者は触る必要はありません。

2

インストール

ダウンロードした .exe をダブルクリック。

⚠ ここで多くの人が止まります

「Windows によって PC が保護されました」 という青い画面が出ます。 「実行しない」ボタンしか見えませんが、慌てないでください。

  1. 左上の小さな 詳細情報 リンクをクリック
  2. すると下に 実行 ボタンが出現するので、それをクリック

※ 無料配布アプリ全般で出る警告で、害はありません。気になる場合は GitHub のソースコード を確認できます。

ウィザードに従ってインストール → スタートメニューから JP225 Monitor 起動。

3

API キーを設定

起動時に設定モーダルが自動で開きます (未設定時)。後から開く時は 右上の ⚙️。

🔑 ここを忘れると AI 機能が動きません

AI 説明 / AI チャット / ニュース翻訳のために、API キーを 1 つ以上設定する必要があります。Gemini が無料で品質も十分。

  1. aistudio.google.com/apikey でキー発行 → コピー
  2. Gemini 欄に 貼り付け (空欄保存は無変更なので注意)
  3. 右下の 保存 をクリック
  4. 状態が 緑になれば完了

※ 詳しい操作は下の「設定画面の使い方」に画面付きで解説。

v0.4 の新機能

v0.4 で変わったこと

価格は OSE mini のリアルタイム配信に切替。24/7 蓄積デーモンと主要レベル表示を新搭載。

価格収集デーモン (24/7 蓄積 + アラート記録)

常駐デーモンが 月 8:45〜土 6:00 (Day 8:45-15:45 / Night 17:00-翌 6:00、休場日除く) に価格を SQLite へ蓄積。モニター起動時に 即ウォームアップ され、メドや z-score 判定がすぐ揃います。v0.5 からはデーモン自身が急変アラートを検知して履歴に記録 するので、アプリを閉じていても 24/7 でアラート履歴が貯まり ます (デーモン稼働中はデーモンが唯一の記録者、アプリは表示のみ・二重記録なし)。設定の「終了」で 「完全終了」にチェック すると閉じる時にデーモンも停止、未チェック (既定) ならバックグラウンドで収集を継続 します。

v0.4 v0.5

主要レベル パネル (意識される水準を点数化)

チャート脇に 上値/下値メド を表示。v0.4.27 で精度を大幅強化:

  • 多スイング・多比率フィボナッチ — 5/10/20セッション＋当日の各スイングから、戻し (23.6/38.2/50/61.8/78.6%) ＋ 拡張 (127.2/161.8%) を描画 (50% ＝ 方向転換ライン ⚑)。
  • 多時間スケールの水準 — 過去セッション高安に加え、前日終値・長期(20セッション)高安・節目 (250/500/1000円) を候補化。
  • 点数化（意識度スコア） — 約25円幅で束ね、重み合計・被テスト回数・合流で採点。複数スイングのフィボや異種水準が重なる「合流帯」を ★★ で最優先表示(日経で非常に効く価格帯)。★＝注目、無印＝通常。代表価格は平均値ではなく 実際に意識される実価格。
  • 🎛️で可変 — 束ね幅・表示本数・選抜窓・合流ボーナス・被テスト係数を詳細パラメータからリビルド不要で調整。

各レベルに 現値からの距離 付き・リアルタイム更新。高さは区切りをドラッグで可変。

v0.4

基礎データ取り込み

N225mini の Excel 履歴 (1 分足) を GitHub 経由でローカル DB に取り込み、メドの履歴を一気に深く できます。設定の 「更新」→「更新をチェック」→「取り込み」。既存の新しいデータは消えず、追記/更新のみ なので安全です。

v0.4

詳細パラメータ (🎛️)

ツールバーの 🎛️ ボタンで開く専用モーダル。Price polling / News polling / Sidecar port / アラート クールダウン (分, 既定 15) に加え、「急変アラート」セクションで急変検知のしきい値を直接調整できます（v0.4.26〜）: 1分/2分急変値幅・加速度しきい値・スコア用値幅・平均変化倍率・急変スコア閾値・急変クールダウン本数・寄り抑制本数・超短期フラッシュ値幅。リビルド不要・保存で即反映し、急変系のしきい値・クールダウン本数・寄り抑制本数は収集デーモンの記録にも自動反映されます（フラッシュ値幅はモニター表示のみ）。

価格・ニュースのポーリングまわりとアラートの再発火間隔を、API キー設定とは別のこのモーダルで管理します。

設定画面の使い方

⚙️ アイコンを押してから API キー保存まで、6 ステップ。

トップバー右端のギアアイコン。初回起動時は API キー未設定なので自動で開きます。

発行ページで AIza... / gsk_... / sk-... をコピー。

JP225 Monitor に戻り、対応欄に貼り付け

「設定済み (上書きする場合のみ入力)」プレースホルダの入力欄に貼り付け。空欄保存は無変更 なので、既存キーを残したい時は触らない。

右下の 保存 をクリック

状態表示が 緑に切り替わる。「閉じる」でモーダル退場。これで AI 説明・チャット・翻訳が即有効化。

保存先: 入力したキーは ~/.jp225-monitor/config.json にローカル保存されます。アプリ外部に送信されることはありません。
polling 設定: 価格・ニュースの取得間隔と port は v0.4 系では ツールバーの 🎛️ 詳細パラメータ モーダルに分離されました (Price polling / News polling / Sidecar port / アラート クールダウン)。詳しくは v0.4 の新機能 を参照。範囲外の値 (例 Price polling=100ms) は reject されます。
更新 (v0.3.18〜 / v0.4 で統合): 設定モーダルの「更新」セクションに現在のバージョンが表示され、「更新をチェック」ボタン1つでアプリ本体と基礎データを同時に確認します。アプリに更新があればその場の「更新」ボタンから DL + 再起動、基礎データに新着があれば「取り込み」ボタンで適用 (起動時の自動トースト通知に加えての手動確認手段)。
基礎データ (v0.4): 「更新」セクションの 「更新をチェック」→「取り込み」 で、N225mini の Excel 履歴 (1 分足) を GitHub 経由でローカル DB に取り込み、主要レベルの履歴を深くできます。既存の新しいデータは消えず追記/更新のみ。
終了 (v0.4): 「終了」セクションの 「完全終了」 チェックを ON にすると、ウィンドウを閉じた時に 24/7 価格収集デーモンも一緒に停止 します。未チェック (既定) なら閉じてもバックグラウンドで価格収集を継続 し、次回起動時に即ウォームアップできます。

相関リーダー

NK225 と最も相関する銘柄を表示。5 分ごとに再評価され、その時々で最も連動性の高い指標がわかる。

価格カード (2 段モメンタム)

監視銘柄をリアルタイム更新。緑=上昇 / 赤=下落、急変時はハイライト点滅。日経の値の上には 「1分」(率 %) と「10秒」(値幅・円) の 2 段モメンタムを表示。「日経225先物」は NIY=F 表記ですが、価格は OSE mini (大阪取引所) のリアルタイム配信 を主軸にしており、~66,000〜67,000 円台の日経指数そのままが出ます。

主要レベル (メド) パネル

価格カード下のパネルに、過去セッション高安を主軸とした 上値/下値メド (重なるレベルは強レベル ★、フィボ 50% は方向転換ライン ⚑、ADR 予測メド も) を現値からの距離付きで表示します。区切りのドラッグで高さ可変。

6
AI チャット入力 + 定型質問

「今の地合いは?」など自由質問。Enter で送信、Shift+Enter で改行。入力欄の上の定型質問ボタン(①②)か番号「1」「2」でもワンタップ質問可。現在の全銘柄価格 + 直近 15 件ニュース + 日経225先物のテクニカル要約が LLM コンテキストに自動付与される。

7
アラート + AI 説明

アラートは日経 225 先物 (NIY=F) 単独で発火 (横断確認なし・リアルタイム)。各行の冒頭に 発生時刻チップ。急変動を検知すると 30 秒以内に AI が要因を 1〜2 文で説明 (同時刻に大きく動いた他資産も材料に)。🔄 で再生成、× で消去。最大 10 件保持。

8
TradingView チャート

右カラム中段に表示。チャート銘柄は FOREXCOM:JP225 (FOREX.com の Japan 225 CFD、JPY 建て) で、参照指数は日経 225 と同一。価格カード「日経225先物」(NIY=F) とほぼ一致します。区切りのドラッグで高さ可変。

9
ニュース (21 ソース)

日本 7 + 英語 14 フィードを 60 秒間隔で集約。英語見出しは [訳] ボタンで日本語化可能。

💡 価格カード「日経225先物」とチャート銘柄の関係
どちらも 同じ日経 225 指数 を見ています。価格カード (OSE mini リアルタイム、NIY=F 表記) とチャート FOREXCOM:JP225 (FOREX.com の JPY 建て Japan 225 CFD) は商品種別こそ違いますが、参照指数は同一・通貨も同じ JPY なので、表示数字も % 変動もほぼ一致します。
機能一覧

できること

急変検知 (価格変化スコア方式)

アラートは日経 225 先物 (NIY=F) 単独で発火。発火条件は次の 3 つ (v0.4.24 で短期/長期の z-score を撤廃):

① 急変 — 完成 1 分足の価格変化(円)のみで判定するスコア方式 (1分変化・2分変化・加速度・平均変化倍率・ブレイク・同方向本数の 6 条件＋急変条件)。クールダウンはバー数方式 (直近発火から 3 本超)。
② 超短期フラッシュ — 5〜10 秒窓で値幅 ≥ 80 円。
③ グランビル — 移動平均(75本)の①転換・②③継続(戻り売り/押し目買い)。

寄り (Day 8:45 / Night 17:00) から 3 本(=最初の 3 分)は全アラートを抑制し、寄付直後の荒れによる誤発火を避けます。②③の共有クールダウンは 既定 15 分 (逆方向は起点越えで解禁、🎛️ 詳細パラメータで変更可)。

ニュース選別の質 (v0.3.9)

金融関連性判定をタイトル+本文ハイブリッドに、LLM へ渡すニュースは急変時刻 ±15 分 → ±60 分 → 4 時間の段階フォールバックで近接優先。的外れな材料引用を抑制。

FAQ

よくある質問

アラート音が鳴りません
ブラウザのセキュリティ仕様で、初回起動時はユーザー操作が必要です。画面右下の 「🔔 サウンドを有効化」 ボタンを 1 回クリックすれば、以降は自動でビープが鳴ります。
「切断」のままで価格が出ない
  1. アプリを完全終了 (タスクトレイにアイコンがあれば右クリック → 終了)
  2. 再起動 (スタートメニューから)
  3. それでもダメなら、最新インストーラで上書きインストール (設定は保持されます)
アラートは出るのに「なぜ動いた」説明が表示されません
API ドットを確認:
  • 全部 = キー未設定 → ⚙️ から 1 つ以上設定
  • 全部 = 全プロバイダ 429 中 → 数分待つ。または別プロバイダ追加
Y (Yahoo) が黄色のまま
Yahoo の quote() (crumb) が拒否されている状態 (短時間に多数アクセスで 429)。価格データは 予備経路 (Yahoo chart API、crumb 不要) が代替するので 画面の価格は引き続き更新 されます。90s → 3min → 5min の段階で自動的に通常経路を再試行。※ v0.3.28 で旧 Investing.com スクレイプ (Cloudflare ブロックで機能不全) から chart API へ置換。

v0.3.7〜v0.3.9 では 1 tick あたり Yahoo を 2 回呼んでいた残骸コードのせいで黄色から戻らない不具合が出ていました。v0.3.10 で単一バルク呼び出しに統合済み — 黄色固定はもう起きません。

価格カードが --- のまま
価格ソース (OSE mini フィード / Yahoo フォールバック) が同時失敗中。ネット接続を確認、数分待つ、ダメなら再起動。
起動時にファイアウォール許可ダイアログ
「アクセスを許可する」で OK。内部 HTTP サーバ (port 3000) のために必要です。プライベートネットワークのみで十分。
SmartScreen 警告は危険ではないですか?
無料配布アプリ全般で出る警告です。ソースコードは GitHub で全公開されているので、不安な場合は読んでから判断してください。「詳細情報」→「実行」で起動できます。
サーバログを見たい (デバッグ用)
トップバーの 📋 ボタンでログモーダルが開きます。直近 200 行を表示、新しいログが自動で下に追加。アプリ再起動で消えます。
polling 間隔 / アラート クールダウンを変えたい
ツールバーの 🎛️ 詳細パラメータ モーダル (v0.4 で ⚙️ から分離)。
  • Price polling (ms)。Yahoo 429 が頻発するなら大きめに
  • News polling (ms, デフォルト 60000)
  • Sidecar port: 1024–65535 (デフォルト 3000)。変更時はアプリ再起動必要
  • アラート クールダウン (分, 既定 15)。鳴りすぎる時は伸ばす (逆方向は起点越えで即解禁)
新版が出たらどうやって知る?
起動 5 秒後に updater が自動チェック。新版があれば右上にトースト通知が出ます。クリック 1 つで DL + 自動再起動 (約 30 秒)。「後で」は 24 時間後再通知。

また、⚙️ 設定画面の「更新」セクションの 「更新をチェック」 ボタンで、いつでも手動でアプリ本体と基礎データの新着を確認できます (v0.3.18〜)。更新があればその場で適用可能。

更新時に「Error opening file for writing」エラーが出る
起動中のサイドカープロセスがファイルロックしているのが原因です。v0.3.3 以降のインストーラ で preinstall フックを追加し、書き込み前に自動 taskkill するので、もう出ません。

v0.3.2 以前を使っていてエラーに当たった場合:

  1. ダイアログで 「中止(A)」 をクリック (※「無視(I)」は絶対押さない — 旧 sidecar.exe が残り起動不能になる)
  2. Ctrl + Shift + Esc でタスクマネージャ → 「詳細」タブ
  3. jp225-sidecar.exe と jp225-monitor.exe を右クリック → タスクの終了
  4. 数秒待ってインストーラを再実行
日経 225 先物の表示価格が想定と違う
日経指数そのままの値 (~66,000〜67,000 円台) が表示されます。v0.4 系では価格は OSE mini (大阪取引所) のリアルタイム配信が主軸で、従来の CME 経由 (約 10 分遅延) ではありません (Yahoo は予備経路)。NIY=F はカード上の表記名として残っています。

参考: ごく初期 (v0.3.7 以前) は NK=F を JPY 換算した ~1 千万円台が出ていましたが、これは Yahoo の古い ALTSYMBOL であり換算も不要だったため、v0.3.8 で撤去済みです。

カードの価格はチャートと数字が違うんですが
ほぼ一致 します。価格カードの「日経225先物」(OSE mini リアルタイム) もチャート (FOREXCOM:JP225) も同じ日経 225 指数を JPY で表示しているため、差は売買スプレッド・取引時間帯の鮮度程度です。

ごく初期 (v0.3.7 以前) のバージョンを使っている場合は古い NK=F × USD/JPY 換算値 (~1 千万円台) になっているので、最新版にアップデートしてください。
