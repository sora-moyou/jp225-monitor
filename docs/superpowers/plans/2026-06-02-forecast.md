# 予測系(ADR予測メド + 時間帯シーズナリティ) Implementation Plan

> REQUIRED SUB-SKILL: superpowers:subagent-driven-development.
**Spec:** `docs/superpowers/specs/2026-06-02-forecast-design.md`
**前提:** `bars_1m`(NIY=F), `getSessionOHLC`(openT付き), `getRecentBars`, `classifySession`,
`isSessionComplete`/`sessionOpenEpoch`(levels.ts, **export 化が必要**), 主要レベル(levelsLoop→computeLevels), chatContext。

## Tasks
1. **forecast.ts ADR**: `levels.ts` から `isSessionComplete` を export 化。`server/forecast.ts` に
   `computeADR(sessions, n, session): {adrUp, adrDown, samples}`(寄り揃い完了セッションの up/downRange の**中央値**、
   セッション種別フィルタ) ＋ `projectTargets(open, adr): {projHigh, projLow}` ＋ tests。
2. **forecast.ts シーズナリティ**: `slotKey(epochMs, slotMin): 'HH:MM'`(JST), `computeSeasonality(bars, slotMin, lookbackSessions?):
   SlotStat[]`(スロットごとに avgReturn%/upRate/avgRange%/samples), `currentAndNextSlot(stats, asOf): {now,next}` ＋ tests。
   (bars は {t, o, h, l, c} 配列。スロット = 同一スロット時刻×日 を集計。)
3. **forecastLoop + route**: `server/loops/forecastLoop.ts`(自前DB, ~120秒ごとに getSessionOHLC + 直近N日 getRecentBars →
   ADR(当日セッション種別) + projectTargets(当日open, 寄り揃い時のみ) + seasonality をキャッシュ。`getForecastSnapshot()`)。
   `server/routes/forecast.ts`(`GET /api/forecast`)。index.ts に route + `startForecastLoop()`。
4. **levels に ADR 注入**: `computeLevels` に任意引数 `extraLevels: {price:number, label:string}[] = []` を追加し候補に混ぜる
   (既存テストは extraLevels 既定[]で不変)。`levelsLoop` が forecast の projHigh/projLow を `[{price:projHigh,label:'ADR上限予測'},
   {price:projLow,label:'ADR下限予測'}]` として渡す(寄り揃い時のみ)。
5. **UI + AI**: `levelsPanel` 上部にシーズナリティ1行(`/api/forecast` を初回取得＋数分ポーリング、現在スロットの
   avgReturn/upRate/avgRange を「13:00台: 平均+0.04% / 上昇60% / 値幅0.18%(N日)」表示)。ADRレベルは levels 結果に
   含まれるので自動表示。`chatContext.buildNikkeiTechnical` に「予測(ADR/シーズナリティ)」ブロックを追記。

各タスクで TDD(該当時)、`npm run typecheck`、関連テスト緑、commit。コード詳細は spec を参照(controller が subagent に渡す)。
Final: `npx vitest run` 全緑、`npm run build:web` 緑。リリース v0.4.20。
