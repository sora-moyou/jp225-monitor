# アラート再設計 spec (v0.6.0)

date: 2026-06-05
status: approved (ユーザー承認 2026-06-05)

## 目的
アラートを「検知器が条件を満たすたびに鳴らす」から、**トレーダーが転換点で問う5つの問いに、根拠付きで、もっともらしい時だけ答える**仕組みへ再設計する。

## 設計原則
1. **2層分離**
   - L1 値動き(急変 shock / フラッシュ slope)= 「大きく動いた」事実。① AI が原因(ファンダ/テクニカル)を判定。
   - L2 テクニカル状態(②の5シグナル)= 反転/継続の局面。共通型に統一し、生成→集約→重複排除→コンフルエンス→1本化 emit。
2. **非イベントは鳴らさない(③)**。否定局面では出さない。出すときは必ず基準値(価格/MA)を文に含め、ユーザーがチャートで最終判断できる。「可能性」表現で断定しない。
3. **精度最優先(④)**。全 L2 シグナルを確定スイングピボット基盤に統一。コンフルエンス・スコアで精度担保。履歴は基準種別×シグナル別の成功定義で実用化。
4. **明確な定型文(⑤)**。「{具体的な基準名}{動作}の可能性」+値併記。

## 共通型
```ts
interface AlertSignal {
  type: 'double' | 'ma_sr' | 'level_sr' | 'break' | 'pivot' | 'trend';
  direction: 'up' | 'down';
  reference: { kind: string; price: number };  // kind: 'sessionLow'|'prevClose'|'ma25'|'swingLow'|...
  stage?: 'forming' | 'confirmed';
  score: number;            // 強度/コンフルエンス
  scoreParts?: string[];    // 内訳(表示/デバッグ)
  text: string;             // 明確な日本語
  triggeredAt: number;
}
```
パイプライン: 各 detector が `AlertSignal[]` を産出 → aggregator が (a) 同一 reference(±tol)を重複排除, (b) 同方向・同局面の整合シグナルを1本に統合してスコア加算, (c) per-type cooldown + score しきい値で間引き → `emitAlert` payload へ写像(detectionKind=type, note=text, level=reference.price, reference_kind/price も記録)。

## ① ファンダ/テクニカル判定
- 急変/フラッシュ発火時、AI 説明のニュース参照窓を「直前の急変アラート時刻(lastShockAt)以降」に限定。
- 関連ニュースあり → ファンダ要因として要約(既存の極性ガード維持)。
- 関連ニュースなし → 「テクニカル要因の可能性(該当ニュースなし)」と明記 + その時点の L2 状態を併記(例「直前に長期高値を上抜け」)。

## ② 5シグナル
1. **double** ダブル天井/大底。swingベースに一本化(micro dtb 廃止)。forming→confirmed。基準=ネック価格+2脚。
2. **ma_sr** MAサポート/レジスタンス(=グランビル継続の改称)。価格がMA(25/75)に接近し**支持/抵抗された**。基準=現値+MA値。
3a. **level_sr** 意識水準で**反発確認後**に支持/抵抗(新規)。基準=水準価格+由来名。⑤例「セッション最安値65,000がサポートされた可能性」。
3b. **break** 水準抜け(v0.5.7整理を継承)。同一水準は 反発(level_sr)か 抜け(break)に出し分け。
4. **pivot** スイング転換点の形成(新規)。確定ピボット発生で通知。**この局面のみ「押し安値/戻り高値」語を許可**。
5. **trend** トレンド転換(グランビル転換 + 単純MAクロス ma を統合)。基準=MA+直近高安。

## ③ 非イベント抑制
- スコア+しきい値。否定(例: 水準を明確に割った→「支持」は出さない)は emit しない。
- 「支持/抵抗された」は**タッチ後に一定幅跳ね返した事実**を要件に(予測でない)。
- 文に必ず基準値。

## ④ 精度 + 履歴/的中率
- コンフルエンス統合(同局面で複数整合→1本・高スコア、単独弱は低スコア/抑制)。
- スコア = 基準の意識度 × 確認の強さ(反発/突破の値幅・速度) × 整合本数 × 鮮度。
- `alerts` に `reference_kind` / `reference_price` 追加。
- シグナル別成功定義(方向正規化): 支持/抵抗=反転継続(割らずに+Y%)、抜け/転換/DTB=順行継続(+Y%/目標到達)。
- **成功定義しきい値はシグナル別に保持**(既定は全シグナル同値=現行15分+0.1%、configStore で変更可)。
- 内訳表示: シグナル種別 × 基準種別の成功率・サンプル数。
- バックテスト・ハーネス: 蓄積barsで過去成功率を計測しきい値を実データ較正。

## ⑤ 明確な説明
- 定型文「{基準名}{動作}の可能性」+値。基準名は具体(セッション最安値/前日Day終値/25MA/長期高/節目)。
- トレンド中立語(v0.5.9)維持、pivot形成のみ方向語可。

## UI
- **パネルは現行維持**(新規状態パネルは作らない)。バナーは既存のテクニカル固定文経路のまま、新 type を plumbing。

## 旧→新
- 残/改称: shock, slope, break, グランビル継続→ma_sr, グランビル転換+ma→trend, swingdtb→double。
- 廃止: micro dtb。
- 新規: level_sr, pivot, コンフルエンス層。
- 旧 detectionKind(dtb/swingdtb/granville/ma)は履歴表示の後方互換で rowKind に残す。

## 確定した設計判断
1. micro dtb 廃止。
2. 支持/抵抗は反発確認後に発火。
3. 成功定義しきい値はシグナル別(既定同値・変更可)。
4. UIパネル現行維持。
5. 完成まで一気通貫、途中判断 leader 一任、不足は v0.6.1。
