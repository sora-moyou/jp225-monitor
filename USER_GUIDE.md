## クイックスタート (3 ステップ)

### 1. ダウンロード

https://github.com/sora-moyou/jp225-monitor/releases/latest

ページ下の **Assets** から `JP225.Monitor_X.X.X_x64-setup.exe` をダウンロード。

### 2. インストール

- ダブルクリック → **「Windows によって PC が保護されました」** で「**詳細情報**」→「**実行**」
- インストールウィザードは既定のままで OK

### 3. API キー設定

起動時に設定モーダルが自動で開きます。

- [Gemini 取得 (無料、推奨)](https://aistudio.google.com/apikey) → `AIza...` をコピー → モーダル「Gemini」欄に貼り付け → 保存
- 詳細: Web 版マニュアル §3

## v0.4 の新機能

- **価格収集デーモン (24/7 蓄積 + アラート記録)** — 常駐デーモンが月 8:45〜土 6:00 (Day 8:45-15:45 / Night 17:00-翌 6:00、休場日除く) に価格を SQLite へ蓄積。モニター起動時に即ウォームアップ。**v0.5 からはデーモン自身が急変アラートを検知して履歴に記録**するので、**アプリを閉じていても 24/7 でアラート履歴が貯まる** (デーモン稼働中はデーモンが唯一の記録者、アプリ側は表示のみで二重記録なし)。設定「終了」の **「完全終了」にチェック** で閉じる時にデーモンも停止、未チェック (既定) ならバックグラウンドで収集継続
- **主要レベル パネル (意識される水準を点数化)** — チャート脇に上値/下値メドを表示 (v0.4.27 で精度強化)。**多スイング・多比率フィボ** (5/10/20セッション＋当日 × 戻し23.6-78.6% ＋ 拡張127.2/161.8%、50%＝方向転換ライン ⚑) ＋ **前日終値/長期(20S)高安/節目(250/500/1000円)** を候補化。約25円幅で束ね、重み・被テスト・合流で**点数化**し、**複数フィボや異種水準が重なる「合流帯」を ★★ で最優先表示**(★＝注目)。代表価格は平均値でなく実際に意識される**実価格**。束ね幅/表示本数/選抜窓/合流ボーナス等は**🎛️でリビルド不要調整**。現値からの距離付き・リアルタイム・高さ可変
- **基礎データ取り込み** — N225mini の Excel 履歴 (1 分足) を GitHub 経由でローカル DB に取り込み、メドの履歴を一気に深く。設定「更新」→「更新をチェック」→「取り込み」。既存データは追記/更新のみ
- **詳細パラメータ (🎛️)** — ツールバーの 🎛️ ボタンで開く専用モーダル。Price polling / News polling / Sidecar port / **アラート クールダウン (分, 既定 15)** に加え、**「急変アラート」セクション**で急変検知のしきい値（1分/2分急変値幅・加速度・スコア値幅・平均変化倍率・急変スコア閾値・急変クールダウン本数・寄り抑制本数・超短期フラッシュ値幅・**グランビル中期/長期MA本数**）を**リビルド不要で即調整** (v0.4.26〜)。急変系のしきい値/クールダウン本数/寄り抑制本数は**収集デーモンの記録にも自動反映**（フラッシュ値幅はモニター表示のみ）
- **UI 文字サイズ可変** — `Ctrl +` / `Ctrl −` で拡大縮小 (`Ctrl 0` でリセット)。チャートは実寸維持
- **アラート表示** — 各行冒頭に発生時刻チップ。最大 10 件ローリング表示、再起動後も保持

## 主な機能

- **リアルタイム銘柄監視** — 価格カード 7 銘柄 (NIY=F / NQ=F / YM=F / ^HSI / CL=F / ^TNX / JPY=X) + バックグラウンドで日経寄与の大きい値がさ株 7 銘柄。日経 225 先物の価格は **OSE mini (大阪取引所) のリアルタイム配信** が主軸 (Yahoo は予備)
- **急変検知 (価格変化スコア方式)** — **アラートは日経225先物 (NIY=F) 単独で発火**。4 条件: ①**急変**(完成1分足の価格変化(円)スコア方式・1分/2分/加速度/平均倍率/ブレイク/同方向の6条件＋急変条件、クールダウンはバー数方式=直近発火から3本超) ②**超短期フラッシュ**(値幅・円, 5〜10秒で80円) ③**グランビル**(中期25・長期75 の2本MA併用、🎛️ で可変) ④**ダブルトップ/ボトム**(**高安関係の水準すべて**を対象・**手前10円**まで接近・左側の**髭タッチ(水準まで手前5円以内に到達)**後の2山目で発火・**ネック割れ不要**・髭がレベルを超えたら不成立。反転狙い。「ダブルトップ/ボトムの可能性あり」と表示。レベルごとに15分クールダウン)。**短期(率%,〜60秒)/長期(5分)の z-score は v0.4.24 で撤廃**。**寄り(Day 8:45/Night 17:00)から3本は全アラート抑制**。**クールダウンを出すのは①急変のみ**(急変が連続表示しないよう自己抑制。既定15分・🎛️ で変更可)。**②③④のテクニカル系はクールダウンを完全無視**(常に発火・他を抑制しない)
- **AI 自動説明** — Gemini → Groq → OpenAI 自動フォールバック。急変時は大きく動いた他資産 (他指数・商品・FX・値がさ株のうち |z|≥4.0) も判断材料に。**グランビル/ダブルトップ・ボトムなどチャートパターン由来のアラートは AI を呼ばず「テクニカル要因」と表示**(ニュース要因ではないため)
- **AI チャット (モニターのデータ参照 + Web 検索)** — 全銘柄価格 + 直近ニュース + 日経225先物テクニカル要約に加え、**このアプリ自身の監視データ (直近アラートの要約・本日のセッション OHLC)** をコンテキストに自由質問。「**なぜ急落?**」には直近の急変を特定して原因分析、種別別の的中率や本日/直近の値動きも即答。手元に無い最新の出来事は **Web 検索 (Gemini の Google 検索グラウンディング)** で出典付きに回答。⚙️の「**Web検索キー**」に Gemini キーを設定 (空欄なら共通の Gemini キーを使用)、モデルは「Web検索モデル」で指定 (既定 `gemini-flash-latest`)。未設定/枠切れでも従来どおり検索なしで回答。ワンタップ定型質問 (① テクニカル分析 ② ファンダメンタル分析 ③ スイング分析) 付き
- **ニュース (RSS + 米経済指標 + 日経総合)** — 日本 7 + 英語 14 の RSS に加え、**米経済指標** (CPI・雇用統計など、結果/予想/前回 + 発表直後の日経225 夜間先物の反応 pt) と **日経225 総合ニュース** をマージ表示。英語見出しはワンクリック翻訳
- **トレードシグナル (表示専用・v0.7.43〜)** — 左上 (主要レベルの上) に、AI エントリー + phase-exit 決済を**紙トラッキング**したシグナルを目立つ表示。エントリー時=`買い78,050指値 / 買い78,500逆指値 ・ 初期LC78,000`、保有中=`● 保有：買い @78,050`、決済時=`✔ 決済79,000`。シグナル発生で**アラート音**(設定で ON/OFF)。**売買命令は送りません** (SSE 現在値で擬似約定するだけ)。右上 **📈** で**履歴 + 収益曲線** (独立保存=再起動でも消えない・設定の「トレードシグナル履歴を消去」で削除)。AI 見送り後の再計画は**節目 (主要レベル) を超えたとき**に行います。jp225-Trade とは完全独立
- **TradingView チャート埋め込み** (日経 225) + 主要レベル パネル
- **音声アラート** — 上昇=高音、下落=低音、初回 1 回手動有効化
- **自動アップデート** — 起動 5 秒後にチェック、トースト通知。設定画面「更新」の「更新をチェック」でアプリ本体と基礎データの新着を手動確認も可能 (v0.3.18〜)
- **レイアウト調整** — パネルの境界をドラッグして各欄の高さ・左右カラムの幅を可変 (左右の中央線はダブルクリックで初期化)。`Ctrl +`/`−` で文字サイズも変更

## トップバーの 4 つの○ (API ステータスドット)

5 秒ごとに自動更新されるデータソース稼働状況:

| ラベル | 意味 |
|---|---|
| **Y** | Yahoo Finance |
| **G** | Gemini |
| **Gr** | Groq |
| **O** | OpenAI |

色: 🟢 正常 / 🟡 待機中 (429) / ⚪ 未設定。hover でカウントダウン表示。

## トラブルシューティング

| 症状 | 対処 |
|---|---|
| アラート音が鳴らない | 画面右下「🔔 サウンドを有効化」を 1 回クリック |
| 「切断」のまま | アプリ完全終了 → 再起動 |
| AI 説明が出ない | ⚙️ で API キー状態確認、未設定なら設定 |
| Y ドットが 🟡 | 価格取得が一時失敗・リトライ中。直近価格は保持(stale)、復帰で緑に戻る |
| 価格カードが `---` | 両ソース失敗、ネット確認 / 再起動 |
| インストール時 "Error opening file" | v0.3.3 以降は自動解消。旧版なら タスクマネージャ で `jp225-sidecar.exe` を終了して再実行 |
| アラートが頻発する | v0.3.7 以前は内部の JPY 換算バグで誤検知が出ていた。v0.3.8 にアップデートで解消 |

## データ保存場所

- 設定 (API キー、polling 間隔、アラート クールダウン等): `%USERPROFILE%\.jp225-monitor\config.json`
- 蓄積価格 DB (24/7 収集デーモン / 基礎データ): `%APPDATA%\jp225-monitor\jp225.db` (node:sqlite, WAL)
- ログ: アプリ内 📋 モーダルのみ (ファイル保存なし)
