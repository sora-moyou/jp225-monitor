API キー設定

最低1つ設定すれば動作。複数設定で 429 時に自動フォールバック。
保存先: ~/.jp225-monitor/config.json (ローカルのみ、送信されません)

AIエントリー
AIエントリー計画(scalp-plan)の上限とバイアスを一元管理します。各レッグの初期LC幅がこの上限を超える計画は自動で外し、バイアスに反する方向は見送りにします(コードで保証)。クールダウンは決済後に次の新規エントリーを一定秒数見送ります(0で無効)。monitorのシグナルエンジンと jp225-Trade の両方に反映されます。
各プロバイダへ極小(1トークン)のテストリクエストを1回ずつ送り、キーが実際に有効かを確認します（消費はごくわずか）。保存済みキーが対象です。
更新
現在のバージョン: …
アプリ本体と基礎データ（履歴1分足）の両方を確認します。基礎データの「取り込み」はローカルDBへ追記/更新のみで、既存の新しいデータは消えません。
終了
未チェック（既定）＝閉じてもバックグラウンドで価格収集を継続。起動時は常に未チェック。
データ
別PCの jp225.db を選んで取り込みます。collector と jp225-Trade を停止し、現行DBをバックアップしてから OR IGNORE で統合し、自動で再起動します。パッケージ版でのみ利用可。
この PC の jp225.db を選んだ場所へ書き出します（別PCへ移してマージする用）。DBは変更せず、停止・再起動もしません。パッケージ版でのみ利用可。
選んだ DB の内容で現在の DB を丸ごと置き換えます（既存データは消去・自動バックアップ後）。2台を完全に同一データにしたい時用。collector/jp225-Trade を停止し自動再起動。パッケージ版でのみ利用可。
トレードシグナル（表示専用）
AIエントリー＋phase-exit決済を紙トラッキングした表示専用のシグナルです。売買命令は送りません。履歴の消去は元に戻せません。

空欄のフィールドは変更されません。設定済みフィールドを置き換えるには新キーを入力。
